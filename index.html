Introducción a la ingeniería de software

La ingeniería de software no solo es desarrollo, también es análisis y documentación. Es un proceso que involucra diferentes métricas de trabajo (tiempos, disciplina, análisis, riesgos, equipo) esto para generar un producto funcional, pero con la misma posibilidad de brindar soporte como garantía. Pasar de propósito general a propósito especifico según el programa.

Introducción a la arquitectura de software

Arquitectura de software es similar al diseño de software. La estructura de sistema se describe en 3 niveles diferenciados por el grado de detalles. El estilo arquitectónico, patrones de diseño y diseño de componentes. Cada una presenta detalles según lo que se quiera observar o encontrar. Los estilos arquitectónicos dependerán de los requerimientos.

Patrones de diseño

Un patrón soluciona un problema de diseño recurrente. Estos patrones se pueden combinar según la magnitud del problema de diseño. Es una solución de diseño a un problema recurrente en un contexto particular.


Diferencia entre un patrón de diseño con estilos arquitectónicos. Los patrones son de medio nivel, pocas clases y relaciones. Los estilos se aplican globalmente en toda la aplicación, no resuelve el problema sino organizar.

API

API (application progamming interface). Establece una vía de comunicación entre softwares y los datos a través de reglas. Hoy en día está basado en peticiones y respuestas.


APIS REST. Es un estilo de arquitectura de software estandarizado usado para desarrollar API ́s se caracteriza por ser simple, estandarizado, escalables, sin estado y de alto rendimiento. Basado en request (solicitud) , response (respuesta). Operaciones de CRUD.
